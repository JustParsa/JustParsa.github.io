RSA Encryption

I used Python to implement the famous RSA encryption algorithm. It can encrypt any input with 32 bit up to 2048 bit encryption. Python was actually a very convenient choice for this project due to the relative ease of working with huge numbers (numbers with 100-300 digits) compared with languages like C++.

The most difficult part of the project was generating huge primes (up to 1024 bit primes!) in a relatively small amount of time. To do this, I would constantly generate random numbers in the necessary bit range and set an upper limit on the maximum number of tries that the program can go through before it's forced to terminate. After every generation of a huge number, I would use the Rabin-Miller probablistic primality test to check if the number was (probably) prime or not. I believe this is one of the best and most efficient primality checks.

At first thought, it might seem as though randomly generating huge numbers and hoping you hit a prime number is a very bad idea and inefficient, but this method works surprisingly well with Rabin-Miller and very rarely is the program forced to terminate. I think on average it takes 1-3 seconds to generate prime numbers for the maximum (2048 bit) encryption.
